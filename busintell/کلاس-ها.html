برنامه‌نویسی شی‌گرا در پایتون (OOP)

برنامه‌نویسی شی‌گرا چیست؟

OOP مخفف Object-Oriented Programming یا برنامه‌نویسی شی‌گرا است.

پایتون یک زبان برنامه‌نویسی شی‌گرا است که به شما امکان می‌دهد کد خود را با استفاده از کلاس‌ها و اشیاء برای سازماندهی بهتر و قابلیت استفاده مجدد ساختار دهید.

مزایای برنامه‌نویسی شی‌گرا

  • ساختار واضحی برای برنامه‌ها فراهم می‌کند
  • کد را برای نگهداری، استفاده مجدد و اشکال‌زدایی آسان‌تر می‌کند
  • به حفظ کد شما بر اساس اصل DRY کمک می‌کند (خودت را تکرار نکن)
  • به شما امکان می‌دهد برنامه‌های قابل استفاده مجدد با کد کمتری بسازید

نکته

اصل DRY به این معنی است که باید از نوشتن کد یکسان بیش از یک بار خودداری کنید. کدهای تکراری را در توابع یا کلاس‌ها قرار دهید و مجدداً از آنها استفاده کنید.

کلاس‌ها و اشیاء چیستند؟

کلاس‌ها و اشیاء دو مفهوم اصلی در برنامه‌نویسی شی‌گرا هستند.

یک کلاس تعریف می‌کند که یک شیء چگونه باید باشد، و یک شیء بر اساس آن کلاس ایجاد می‌شود. برای مثال:

کلاس اشیاء
میوه سیب، موز، انبه
ماشین ولوو، آئودی، تویوتا

وقتی یک شیء از یک کلاس ایجاد می‌کنید، تمام متغیرها و توابع تعریف شده در داخل آن کلاس را به ارث می‌برد.

تست درک مطلب: برنامه‌نویسی شی‌گرا

کدام یک از موارد زیر مزیت اصلی برنامه‌نویسی شی‌گرا نیست؟

کلاس‌ها و اشیاء در پایتون

پایتون یک زبان برنامه‌نویسی شی‌گرا است.

تقریباً همه چیز در پایتون یک شیء است، با ویژگی‌ها و متدهای خود.

یک کلاس مانند یک سازنده شیء یا یک "الگو" برای ایجاد اشیاء است.

ایجاد یک کلاس

برای ایجاد یک کلاس، از کلمه کلیدی class استفاده کنید:

مثال

یک کلاس به نام MyClass با ویژگی x ایجاد کنید:

class MyClass:
  x = 5

ایجاد شیء

حالا می‌توانیم از کلاس MyClass برای ایجاد اشیاء استفاده کنیم:

مثال

یک شیء به نام p1 ایجاد کنید و مقدار x را چاپ کنید:

p1 = MyClass()
print(p1.x)

حذف اشیاء

می‌توانید اشیاء را با استفاده از کلمه کلیدی del حذف کنید:

مثال

شیء p1 را حذف کنید:

del p1

اشیاء متعدد

می‌توانید چندین شیء از یک کلاس ایجاد کنید:

مثال

سه شیء از کلاس MyClass ایجاد کنید:

p1 = MyClass()
p2 = MyClass()
p3 = MyClass()

print(p1.x)
print(p2.x)
print(p3.x)

توجه

هر شیء مستقل است و کپی خود را از ویژگی‌های کلاس دارد.

تست درک مطلب: کلاس‌ها و اشیاء

کدام یک از گزینه‌های زیر در مورد کلاس‌ها و اشیاء در پایتون صحیح است؟

متد __init__()

همه کلاس‌ها یک متد داخلی به نام __init__() دارند که همیشه هنگام راه‌اندازی کلاس اجرا می‌شود.

متد __init__() برای اختصاص مقادیر به ویژگی‌های شیء یا انجام عملیات‌های لازم هنگام ایجاد شیء استفاده می‌شود.

مثال

یک کلاس به نام Person ایجاد کنید، از متد __init__() برای اختصاص مقادیر برای نام و سن استفاده کنید:

class Person:
  def __init__(self, name, age):
    self.name = name
    self.age = age

p1 = Person("امیل", 36)

print(p1.name)
print(p1.age)

توجه

متد __init__() به طور خودکار هر بار که از کلاس برای ایجاد یک شیء جدید استفاده می‌شود، فراخوانی می‌شود.

چرا از __init__() استفاده کنیم؟

بدون متد __init__()، باید ویژگی‌ها را به صورت دستی برای هر شیء تنظیم کنید:

مثال

یک کلاس بدون __init__() ایجاد کنید:

class Person:
  pass

p1 = Person()
p1.name = "توبیاس"
p1.age = 25

print(p1.name)
print(p1.age)

استفاده از __init__() ایجاد اشیاء با مقادیر اولیه را آسان‌تر می‌کند:

مثال

با __init__()، می‌توانید مقادیر اولیه را هنگام ایجاد شیء تنظیم کنید:

class Person:
  def __init__(self, name, age):
    self.name = name
    self.age = age

p1 = Person("لینوس", 28)

print(p1.name)
print(p1.age)

متد __init__() در پایتون چه زمانی اجرا می‌شود؟

وراثت در پایتون

وراثت به ما امکان می‌دهد کلاسی را تعریف کنیم که تمام متدها و ویژگی‌ها را از کلاس دیگری به ارث می‌برد.

کلاس والد، کلاسی است که از آن ارث‌بری می‌شود، که به آن کلاس پایه نیز می‌گویند.

کلاس فرزند، کلاسی است که از کلاس دیگری ارث‌بری می‌کند، که به آن کلاس مشتق شده نیز می‌گویند.

ایجاد یک کلاس والد

هر کلاسی می‌تواند یک کلاس والد باشد، بنابراین نحو آن مانند ایجاد هر کلاس دیگری است:

مثال

یک کلاس به نام Person با ویژگی‌های firstname و lastname و یک متد printname ایجاد کنید:

class Person:
  def __init__(self, fname, lname):
    self.firstname = fname
    self.lastname = lname

  def printname(self):
    print(self.firstname, self.lastname)

# از کلاس Person برای ایجاد یک شیء استفاده کنید و سپس متد printname را اجرا کنید:

x = Person("جان", "دو")
x.printname()

ایجاد یک کلاس فرزند

برای ایجاد کلاسی که قابلیت‌های کلاس دیگری را به ارث می‌برد، کلاس والد را به عنوان پارامتر هنگام ایجاد کلاس فرزند ارسال کنید:

مثال

یک کلاس به نام Student ایجاد کنید که ویژگی‌ها و متدهای کلاس Person را به ارث می‌برد:

class Student(Person):
  pass

توجه

از کلمه کلیدی pass استفاده کنید وقتی نمی‌خواهید ویژگی‌ها یا متدهای دیگری به کلاس اضافه کنید.

اکنون کلاس Student همان ویژگی‌ها و متدهای کلاس Person را دارد.

مثال

از کلاس Student برای ایجاد یک شیء استفاده کنید و سپس متد printname را اجرا کنید:

x = Student("مایک", "اولسن")
x.printname()
